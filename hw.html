Data description¶

Link to data set could be found here.

Data set consists of the most popular movies according to IMDB - the world's most popular and authoritative source for movie, TV and celebrity content. Data set was collected in period between 2006 and 2016 years. Total size of data set is 1000 entities. To meet the requirements we've choosen movies from 2006 to 2013 years and the final size of data set is 438 records. We've also chosen several features to work with, our feature set includes:

  1. Genre - genre of the movie, categorical feature (Adventure, Mystery, Sci-Fi, etc.)
  2. Year - the year of movie release, rank feature (from 2006 to 2013)
  3. Runtime - length of the movie in minutes, numerical feature
  4. Rating - actual rating of the movie according to IMDB, numerical feature (from 0 to 10)
  5. Votes - total amount of votes, numerical feature
  6. Revenue - proceeds of the movie in millions USD, numerical feature
  7. Metascore - average score, which includes critics' scores, numerical feature (from 0 to 100)

The main motivation behind this data set is that we both are interested in movies and it would be very fastinatingly to find the correlations between features, find which role plays some feature, which models we would be able to build upon this data set. The final and the most important thing is to increase our knowledge about the moviemaking bussiness through data exploration. In the Data preprocessing section one may find examples of preprocessed data set.

Feature metascore is being compared between clusters #1 and #3. Here, the main hypothesis $H_0: \mu_1 = \mu_2$ vs alternative $H_1: \mu_1 \ne \mu_2$. The algorithm is as it was in the lecture: bootstrap distributions of K trials in cluster #1 and #3 will be computed. Afterwards, confidense intervals for differences of means will be computed. If 0 is in interval, then $H_0$ is accepted, otherwise, rejected.
